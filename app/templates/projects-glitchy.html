glitchy

glitchy is a short Python script that effectively "glitches" jpegs. It works by copying and pasting a random portion of the jpg data to random locations a random number of times.

the code can be found here

due to the randomness of the script, some images will be glitched to corruption. This means the browser will not properly display the image, and the server will return some type of error. This happens when the header data of the jpeg file is overwritten. I'm working on a way to avoid copying/pasting over the header data of the jpeg file which will resolve the issue.

until I work out the kinks, some ways to avoid a corrupted image are:

  • use medium or larger images.

    small images contain way less data, which means there is a higher chance of the header data being overwritten

  • don't try to glitch an image too many times.

    this will increase the chances of overwriting the header data
